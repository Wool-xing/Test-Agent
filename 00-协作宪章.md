## §0 · 双北极星（Test-Agent 特化）

### §0a · 落地准则（双尺平行）

- **0a-1 短期**：新手 30 分钟内 clone + install + `tagent demo` 跑通
- **0a-2 终态**：4 层闭环 + agent 协作 + skill 调用 + 真 LLM-driven agent 跑通

任何决策都用这两把尺子量。短期尺为可被立刻验证的最小起点；终态尺为最终交付目标，防止"修到能 install 就停"。

反向约束直接继承通用宪章 §0a。

### §0b · 诚实准则

替代通用宪章模板中的"安全准则"——Test-Agent 几乎无运行时攻击面（不是 web 应用），真正在乎的质量维度=诚实。

四层覆盖：

- **静态层**：文档零虚假承诺 / 零私域路径泄漏 / 零未脱敏 PII / 零未实测数字
- **运行层**：跑命令产出的日志 / 报告 / HANDOFF 不含未脱敏路径 / 不抄文档当实测 / baseline 误判须自承
- **架构层**：默认 `README` / `NOTICE.md` / `SECURITY.md` / `CODE_OF_CONDUCT.md` / `CONTRIBUTING.md` 等交付物不引用 `FULL_GUIDE.md` 愿景话术（见 §4 角色分离）
- **流程层**：每个 fix 完成后扫描"是否引入新虚假承诺 / 数字膨胀 / 法律风险 / 路径泄漏"

### 修复优先级 · 三尺子取最大值

`priority = max(P_严重度, 0a_落地链卡点权重, 0b_诚实违反严重度权重)`

#### P 严重度
- **P0**：阻断 0a-1 短期尺（新手 install/demo 跑不起来）
- **P1**：阻断 0a-2 终态尺（agent 协作 / skill 调用断链）
- **P2**：影响用户理解或信任（虚假承诺、数字误差、路径泄漏）
- **P3**：格式 / 措辞 / 死链等微伤

#### 0a 落地链卡点权重
- **阻断式**：跟着文档走会卡在某步, 无法继续
- **显著影响**：文档说能做 A, 实测 A 部分可用 / 需绕路
- **轻微**：表述不清但可推断
- **不影响**：纯内部文档

#### 0b 诚实违反严重度（暴露面 × 引用辐射 × 用户可见度）
- **暴露面**：虚假承诺 > 数字误差 > 路径泄漏 > 法律措辞瑕疵
- **引用辐射数**：grep 出多少其他文件引用该 finding 所在节（>10 = 高 / 3-10 = 中 / <3 = 低）
- **用户可见度**：`README` / `启动指南` / `NOTICE` 级 > `02-专家定义` / `03-技能定义` 级 > 内部 `INDEX.md` 级

## §1.3 · 六道闸 Test-Agent 文档语义

通用机制不复述（见元宪章 §1.3）。本节只填项目特化定义。

| 闸 | Test-Agent 取值 |
|---|---|
| a · 静态 | markdown lint + 死链检查 + 中英文混排空格规范 |
| b · 副作用 | grep 改动符号 / 节标题 / 文件名的所有引用（通用） |
| c · 跨层契约 | `README.md` ↔ `NOTICE.md` ↔ `SECURITY.md` ↔ `CODE_OF_CONDUCT.md` ↔ `CONTRIBUTING.md` ↔ `FULL_GUIDE.md` ↔ `00-项目导航.md` 一致性 |
| d · 实测 | **真跑命令验证，不抄文档**。声明"30 秒启动" → 实测计时；声明"支持 X" → 实跑确认；声明数字 → 给出 commit/测试 ID |
| e · 回归 | 跑所有已修 finding 的"防回归检查点"（通用） |
| **f · 诚实自检** | **替代元宪章 f 安全自检, 见下子项** |

### f 闸 · 诚实自检子项

| 子项 | 检查 |
|---|---|
| f1 · 数字证据 | 改动里所有数字（耗时 / 行数 / 文件数 / 覆盖率 / 用户数）必带证据：commit ID / test run ID / 实测命令 + 输出, 否则删除该数字 |
| f2 · 路径泄漏 | grep 私域路径 (`D:\项目文件\` / `_精髓库` / `灵感库` / `~/.claude/projects/`)；公开仓库文档命中 = 必删 |
| f3 · 虚假承诺不回潜 | 已删的虚假承诺（"已支持 X" 实际未实现）扫描是否被新写入；同语义换说法也算回潜 |
| f4 · 法律措辞合规 | 检查 LICENSE / NOTICE / SECURITY 引用第三方时, 引用方式合规, 不夸大背书 |

f 闸的产物追加到 `findings 进度表` 的"防回归检查点"登记表（与 e 闸共表, **类别列区分**）。

## §2 · 反馈模板占位符取值

通用模板见元宪章 §2。本节定义 Test-Agent 内每个 `{{项目特化}}` 的具体值。

| 占位符 | Test-Agent 取值 |
|---|---|
| `{{项目特化: a 静态}}` | markdown lint 输出 + 死链扫描结果 |
| `{{项目特化: c 跨层契约}}` | README/NOTICE/SECURITY/CODE_OF_CONDUCT/CONTRIBUTING/FULL_GUIDE/00-项目导航 互相 grep 一致性 |
| `{{项目特化: d 实测}}` | 实跑命令 + 输出截取（不抄文档） |
| `{{项目特化: f 0b 自检}}` | f1 数字证据 / f2 路径泄漏 / f3 虚假承诺不回潜 / f4 法律措辞合规 四子项结果 |
| `{{项目特化视角}}` | **数字证据视角**：修前（数字 X 无来源 / 来自虚报）→ 修后（数字 X 实测于 commit Y / 测试 ID Z） |

### 反馈模板示例（Test-Agent 完整版）

```
## Fix [bundle-1-#3] · [问题标题]

### 自我反问（机制 1.1）
- Q1 问题根因：[根因 + 当前 fix 是否触达]
- Q2 关联影响：[波及文件清单]
- 范围决策：[修当前层 / 顺路 / 下一个 fix]

### 改动
- README.md:42 + NOTICE.md:18 改了什么

### 6 道闸验证
- a 静态：markdown lint 0 警告 / 死链 0
- b 副作用：grep "X" → 命中 3 处, 已逐个 verify
- c 跨层契约：README/NOTICE/SECURITY/FULL_GUIDE/00-项目导航 一致
- d 实测：实跑 `tagent demo` → 输出 [截取]（不抄文档）
- e 回归：已修 finding 23 个检查点全过
- f 诚实自检：f1 数字带 commit ID a3b5c2 / f2 grep _精髓库 0 / f3 已删承诺无回潜 / f4 NOTICE 引用合规

### 防回归检查点（写入 findings 进度表）
- `grep -n "30 秒启动" README.md` → 必须 0 行 · 类别：f3 虚假承诺
- `grep -rn "D:\\项目文件\\_精髓库" .` → 必须 0 行 · 类别：f2 路径泄漏

### 落地准则评估（0a）
- 0a-1 解锁：新手 install 文档现可被相信（删了虚假"30 秒"承诺）
- 0a-2 解锁：未涉及, 下一节处理

### 诚实准则评估（0b）
- 关闭 0b 违反面：虚假承诺 ×1 / 路径泄漏 ×2

### 数字证据视角
- 修前：声明"30 秒启动"无来源, baseline 实测 47 秒（commit a3b5c2 实测）
- 修后：改写"约 1 分钟启动（实测 47 秒, commit a3b5c2）"

### 全局视角
- 影响面：README 之外 NOTICE/启动指南 同句承诺已同步删
- 顺路优化点（不动手）：CHANGELOG 提及"30 秒"也需扫, 列下一 fix

### HANDOFF
- 当前 fix 状态：完成
- 未完事项：CHANGELOG / 部署说明 同句扫除（next fix）
```

## §4 · 角色分离 · Test-Agent 交付物清单

**禁引用宪章愿景话术的交付物**（面向公开仓库用户）：

- `README.md`
- `NOTICE.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- `CONTRIBUTING.md`
- `LICENSE`
- `CHANGELOG.md`
- `01-快速开始/` 全部文件（使用手册 / 部署说明 / 配置清单 / 交付物清单）
- `02-专家定义/README.md`
- `03-技能定义/README.md`
- `04-配置文件/INDEX.md`
- `00-项目导航.md`

**禁止引用的具体话术**（FULL_GUIDE 内部哲学）：

- "三公理 + 五条铭文"
- "灵魂底色"
- "九大簇"
- "认知暗物质层"
- "测试热寂与熵减祭司"
- "未来僧侣阶层"
- "簇 N · ___"系命名
- "维度全图"
- 任何 FULL_GUIDE §🌌 / §🎭 段落的内部代号引用

**允许引用**：
- 协作机制（"按宪章 §1.3 跑 6 道闸"）
- 工程概念（"七阶段工作流"）
- 具体 utils / skill / agent 名称

**理由**：交付物面向公开仓库用户和开源社区, 引用愿景=用户被迫吞下项目内部哲学。FULL_GUIDE 是愿景白皮书自留地, 不外溢。

## §5 · 修复优先级算法 · Test-Agent 尺子

通用公式：`priority = max(P_严重度, 0a_落地链卡点权重, 0b_诚实违反严重度权重)`

### 决策示例

| finding 类型 | P | 0a 卡点 | 0b 违反 | 取大 = priority |
|---|---|---|---|---|
| README 写"已支持 Java, 实际未实现" | P2 | 显著影响 | 虚假承诺 × README 级 × 高辐射 | **0b 高 = 最优先** |
| 02-专家定义/03 内部 typo | P3 | 不影响 | 不违反 | **P3 = 低** |
| 启动指南有死链卡新手 | P0 | 阻断式 | 不违反 | **P0 / 0a 阻断 = 最优先** |
| FULL_GUIDE 内部章节 typo | P3 | 不影响 | 不违反 | **P3 = 低** |
| NOTICE 把第三方 license 写错 | P1 | 不影响 | 法律措辞 × NOTICE 级 | **0b = 高优先** |

## §6 · Test-Agent 特化红线

通用红线（凭据零写入 / 反馈不造假 / 用户未拍板不擅自推进）直接继承。

**Test-Agent 追加**：

1. **零虚假承诺**：未实测的数字 / 未跑通的 demo / 未实现的功能, 不写"已支持"。说"未支持"或"路线图 Phase N"。
2. **零路径泄漏**：私域路径 (`D:\项目文件\_精髓库\` / `D:\项目文件\灵感库\` / `~/.claude/projects/`) 不进 public 仓库文档。
3. **HANDOFF 必带**：会话结束未完事项必交班, 不假装完成。续会话首步必读 HANDOFF。
4. **角色分离强制**：交付物（见 §4 清单）引用 FULL_GUIDE 愿景话术 = 违反, 立即回正。
5. **数字铺平专条**：批量修复数字误差（baseline 误判）时, 反馈仍列每条改动出处, 不"批量过"。
