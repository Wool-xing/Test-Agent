## 测试设计方法

### 核心方法组合

| 方法 | 适用场景 | 优先级 |
|------|---------|-------|
| 等价类划分 | 输入验证、状态转换 | 必用 |
| 边界值分析 | 数值范围、字符长度 | 必用 |
| 判定表 | 多条件组合逻辑 | 复杂业务 |
| 场景法 | 业务流程、用户故事 | 必用 |
| 错误推测法 | 历史缺陷区域 | 重点模块 |
| 探索性测试 | 新功能、快速验证 | 补充 |
| 正交实验法 | 参数组合爆炸 | 性能/兼容 |

### 优先级分级标准（占比合计 = 100%）

| 优先级 | 标准 | 占比 |
|--------|------|------|
| P0 | 核心业务路径，阻塞上线 | 10% |
| P1 | 主要功能，影响主流程 | 30% |
| P2 | 次要功能，影响体验 | 40% |
| P3 | 边缘场景，体验优化 | 20% |

## 用例设计原则

1. **独立性**：每条用例独立可执行，不依赖其他用例的执行结果
2. **原子性**：一条用例只验证一个功能点
3. **可重复性**：用例可以在任何时间多次执行并得到相同结果
4. **可追溯性**：用例与需求点一一对应
5. **完整性**：覆盖正向、反向、边界、异常四类场景

## 用例 ID 规范

格式：`TC-{MODULE}-{TYPE}-{NUM}`

- MODULE：模块大写英文（如 LOGIN、PAYMENT、ORDER）
- TYPE：UI / API / PERF / SEC（用例类型）
- NUM：3 位数字递增

示例：`TC-LOGIN-UI-001`、`TC-LOGIN-API-001`、`TC-PAYMENT-PERF-001`

## 标准用例字段

| 字段 | 说明 | 通用 / API |
|------|------|----------|
| 用例ID | 见上规范 | 通用 |
| 模块 | 所属功能模块 | 通用 |
| 类型 | UI / API / PERF / SEC | 通用 |
| 优先级 | P0/P1/P2/P3 | 通用 |
| 用例名称 | 简洁描述验证点 | 通用 |
| 前置条件 | 执行前提 | 通用 |
| 测试步骤 | 1.步骤1 2.步骤2 | 通用 |
| 测试数据 | 输入数据 | 通用 |
| 预期结果 | 明确可验证 | 通用 |
| Method | GET/POST/PUT/DELETE | 仅 API |
| Path | /api/v1/... | 仅 API |
| Headers | Content-Type 等 | 仅 API |
| ExpectedStatus | 200/401/... | 仅 API |
| 实际结果 | 执行时填写 | 通用 |
| 执行状态 | Pass/Fail/Skip/Block | 通用 |
| 备注 | 关联Bug/特殊说明 | 通用 |

## 场景覆盖检查表

设计完成后，逐项验证覆盖情况：

```
功能覆盖：
✅ 核心 Happy Path（正向主流程）
✅ 替代路径（其他有效操作方式）
✅ 异常路径（错误处理）
✅ 边界条件（最大、最小、临界值）
✅ 权限控制（不同角色的访问权限）
✅ 并发场景（同一资源并发操作）
✅ 数据状态（各种数据状态下的行为）

非功能覆盖：
✅ 性能基线（响应时间验证）
✅ 安全验证（SQL注入、XSS等）
✅ 兼容性（主流浏览器/设备）
✅ 国际化（如有多语言）
```

## Excel 输出（4 Sheet 标准）

由 `utils/excel_generator.create_testcase_excel(testcases, output_path)` 生成：

```
Sheet1 用例总览     - 优先级分布 + 模块分布统计
Sheet2 测试用例     - 完整字段（16 列，含 API 字段）
Sheet3 P0冒烟集     - 仅 P0 用例（带前置条件、数据）
Sheet4 P0_P1回归集  - P0+P1 用例
```

落盘路径：`workspace/测试用例/testcases_{模块}_{YYYYMMDD}.xlsx`

## 登录模块示例

```python
LOGIN_TESTCASES = [
    # P0 核心
    {"id": "TC-LOGIN-UI-001", "module": "登录", "type": "UI", "priority": "P0",
     "name": "正确账号密码登录",
     "steps": ["打开登录页", "输入正确账号", "输入正确密码", "点击登录"],
     "expected": "跳转首页，显示用户名"},

    {"id": "TC-LOGIN-API-001", "module": "登录", "type": "API", "priority": "P0",
     "name": "登录接口返回 token",
     "method": "POST", "path": "/api/v1/auth/login",
     "headers": "Content-Type: application/json",
     "data": '{"username":"testuser","password":"Test@123456"}',
     "expected_status": 200,
     "expected": "返回 token 字段非空"},

    # P1 异常
    {"id": "TC-LOGIN-UI-002", "module": "登录", "type": "UI", "priority": "P1",
     "name": "密码错误提示",
     "data": "正确账号+错误密码",
     "expected": "显示'账号或密码错误'，不暴露账号是否存在"},

    {"id": "TC-LOGIN-UI-003", "module": "登录", "type": "UI", "priority": "P1",
     "name": "连续失败5次锁定",
     "steps": ["连续输错密码5次"],
     "expected": "账号锁定，提示锁定时间"},

    # P2 安全
    {"id": "TC-LOGIN-SEC-001", "module": "登录", "type": "SEC", "priority": "P2",
     "name": "SQL 注入防护",
     "data": "账号: admin' OR '1'='1",
     "expected": "登录失败，无异常错误，无 SQL 报错回显"},
]
```

## 调用示例

```python
from utils.excel_generator import create_testcase_excel

create_testcase_excel(
    LOGIN_TESTCASES,
    f"workspace/测试用例/testcases_登录_{datetime.now():%Y%m%d}.xlsx"
)
```

## 协作输出

用例设计完成后，提供给：

- **test-executor**：完整用例 Excel + 执行顺序建议
- **test-lead**：覆盖率分析 + 风险提示
