## Page Object 模板（Android）

```python
# mobile/pages/android/login_page.py
from appium.webdriver.common.appiumby import AppiumBy


class LoginPageAndroid:
    def __init__(self, driver):
        self.driver = driver
        self.username = (AppiumBy.ID, "com.example.app:id/et_username")
        self.password = (AppiumBy.ID, "com.example.app:id/et_password")
        self.login_btn = (AppiumBy.ID, "com.example.app:id/btn_login")
        self.error_msg = (AppiumBy.ID, "com.example.app:id/tv_error")

    def login(self, user: str, pwd: str):
        self.driver.find_element(*self.username).send_keys(user)
        self.driver.find_element(*self.password).send_keys(pwd)
        self.driver.find_element(*self.login_btn).click()

    def get_error(self) -> str:
        return self.driver.find_element(*self.error_msg).text
```

## Android Monkey 稳定性测试（utils.mobile_driver.run_monkey）

随机事件注入压测，发现 Crash / ANR / 内存泄漏。

### 命令行调用

```bash
# 基础用法：1 万事件，间隔 200ms
python -m utils.mobile_driver monkey \
    --package com.example.app \
    --events 10000 \
    --throttle 200 \
    --seed 12345 \
    --output workspace/执行日志/monkey/

# 长时压测：10 万事件 + 多设备 serial 指定
python -m utils.mobile_driver monkey \
    --package com.example.app \
    --events 100000 \
    --throttle 100 \
    --serial emulator-5554 \
    --timeout 7200
```

### Python API（更细粒度控制事件类型分布）

```python
from utils.mobile_driver import run_monkey

result = run_monkey(
    package="com.example.app",
    event_count=10000,
    throttle_ms=200,
    seed=12345,                         # 可重放种子
    pct_touch=40,                       # 触摸 40%
    pct_motion=25,                      # 滑动 25%
    pct_nav=15,                         # 方向键 15%
    pct_majornav=10,                    # 主要导航键
    pct_syskeys=5,                      # 系统按键
    pct_appswitch=2,                    # 应用切换
    pct_anyevent=3,                     # 其他事件
    output_dir="workspace/执行日志/monkey",
)
# result: {"event_count", "exit_code", "crashes", "anrs", "duration_sec", "stable", "log_file"}

assert result["stable"], f"稳定性测试失败：crash={result['crashes']}, anr={result['anrs']}"
```

### Monkey 测试集成到 pytest

```python
import pytest
from utils.mobile_driver import run_monkey, archive_logcat


@pytest.mark.p1
@pytest.mark.mobile
@pytest.mark.android
@pytest.mark.stability
def test_monkey_stability():
    """TC-MOBILE-AND-STAB-001: 稳定性 Monkey 1 万事件"""
    result = run_monkey(
        package="com.example.app",
        event_count=10000,
        throttle_ms=200,
        seed=42,
    )
    assert result["crashes"] == 0, f"发现 {result['crashes']} 次崩溃，详见 {result['log_file']}"
    assert result["anrs"] == 0, f"发现 {result['anrs']} 次 ANR"
```

### Monkey 门禁建议

| 指标 | 要求 |
|------|------|
| Crash 数 | = 0 |
| ANR 数 | = 0 |
| 完成事件数 | ≥ 设定 event_count |
| 崩溃率（crash / event_count） | < 0.01% |

### Crash 复现

monkey 失败时 `result.seed` 记录种子。同 seed 同 throttle 可重放：

```bash
python -m utils.mobile_driver monkey \
    --package com.example.app \
    --events 10000 \
    --seed 42 \
    --throttle 200
```

### 与 logcat 协同

run_monkey 自动调 `archive_logcat()` 归档 crash 时刻的 Android 日志到 `workspace/执行日志/logcat/`。
Bug 提交时附带 monkey log + logcat 两份给开发。

## 与其他 agent 协作

- **testcase-designer**：移动端用例标注 type=MOBILE，子类型 ANDROID/IOS/WX
- **automation-engineer**：协调脚本生成（本 agent 专注移动端，automation-engineer 专注 Web/API）

- **bug-manager**：移动 Bug 含设备型号/系统版本/截图/logcat

| 字段 | 必填 | 示例 |
|------|------|------|
