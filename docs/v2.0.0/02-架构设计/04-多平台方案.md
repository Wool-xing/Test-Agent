## 2. 平台差异与适配策略

### 2.1 路径处理

**问题:** Windows 使用 `\` 分隔符，macOS/Linux 使用 `/`。

**策略:** 全项目统一使用 `pathlib.Path`，禁止字符串拼接路径。

**实现位置:** `utils/paths.py`

```python
# 核心路径工具 — 全平台兼容
from pathlib import Path

def get_output_base(project: str | None = None) -> Path:
    """workspace/测试报告/{PROJECT_NAME}  — 中文locale自动适配"""
    custom = os.getenv("TAGENT_OUTPUT_DIR")
    if custom:
        return Path(custom) / project
    default = "workspace/测试报告" if _is_chinese_locale() else "workspace/test-reports"
    return Path(default) / project

def get_output_dir(sub_path: str = "", run_id: str | None = None) -> Path:
    """workspace/测试报告/{PROJECT}[/{run_id}]/{sub_path}"""
    ...
```

**关键原则:**
- 内部统一 `pathlib.Path`，仅在系统调用边界（subprocess、SSH）转换为字符串
- `Path(...) / "sub"` 语法自动处理分隔符
- 所有 `open()` 使用 `Path.read_text()` / `Path.write_text()`
- CI 环境变量 `RUNNER_OS` 不用于路径决策（用 `sys.platform` 或 `os.name`）

### 2.2 行尾符处理

**策略:** 仓库统一 LF，`.gitattributes` 强制。运行时按需转换。

```gitattributes
# .gitattributes
* text=auto eol=lf
*.py text eol=lf
*.md text eol=lf
*.yaml text eol=lf
*.json text eol=lf
*.bat text eol=crlf
*.ps1 text eol=crlf
```

**运行时处理:**
- Python `open()` 默认 `newline=None` 即通用换行模式
- subprocess 输出使用 `text=True` + `universal_newlines=True`
- 写入文件始终使用 `
`（Python 在 Windows 上自动转 `

` 当 `newline=None`）

### 2.3 进程管理差异

| 操作 | Windows | macOS/Linux | 统一方案 |
|------|---------|------------|---------|
| 创建子进程 | `CreateProcess` | `fork/exec` | `subprocess.Popen` / `asyncio.create_subprocess_exec` |
| 进程信号 | 无SIGALRM/SIGTERM | POSIX信号完整 | `threading.Timer` 模拟超时；`proc.kill()` = `TerminateProcess` |
| 进程组 | 不支持 | `os.setsid` | 仅L1沙箱需要；`runtime/core/sandbox.py` 按平台分支 |
| 资源限制 | Job Object API | `resource.setrlimit` | SandboxConfig 统一配置，底层按平台实现 |
| PATH查找 | `where` | `which` | `shutil.which()` 全平台统一 |

**`runtime/infra/timeout.py` 中的平台适配:**
```python
def timeout_guard(seconds: float, label: str = "operation"):
    """跨平台超时上下文管理器"""
    if sys.platform == "win32":
        # Windows: 使用 threading.Timer 模拟 SIGALRM
        ...
    else:
        # Unix: 使用 signal.SIGALRM
        signal.signal(signal.SIGALRM, _handler)
        signal.alarm(int(seconds))
```

### 2.4 Shell 与命令执行

**问题:** Windows 无 bash，`cmd.exe` 语法不同。

**策略:**
- 内部命令使用 `subprocess.Popen(argv, ...)` 列表形式，绕过 shell
- 需要 bash 的场景使用 `runtime/backends/docker.py` 容器内执行
- LLM 生成的命令先经 `shlex.split()` 解析再执行
- 用户自定义脚本通过 `#!/usr/bin/env bash` shebang + Git Bash / WSL 运行

### 2.5 文件权限

| 平台 | 特性 | 处理 |
|------|------|------|
| Linux/macOS | `chmod`, `chown`, umask | 沙箱创建时设置 `work_dir.chmod(0o700)` |
| Windows | ACL, 只读属性 | `os.chmod` 有限支持；忽略权限错误 |
| 所有平台 | 临时文件 | `tempfile.mkdtemp` 自动设置安全权限 |

### 2.6 环境变量

| 场景 | Windows | macOS/Linux | 统一 |
|------|---------|------------|------|
| 分隔符 | `;` | `:` | `os.pathsep` |
| 大小写 | 不敏感 | 敏感 | 统一大写（`TAGENT_*`） |
| 换行 | CRLF | LF | 读取后 `.strip()` |

## 3. 平台隔离架构

### 3.1 隔离原则

业务逻辑中禁止出现以下模式：
```python
# 禁止: 在业务代码中散落平台判断
if sys.platform == "win32":
    path = path.replace("/", "\\")
elif sys.platform == "darwin":
    path = path.replace("\\", "/")

# 正确: 在适配层集中处理
from utils.platforms import normalize_path
path = normalize_path(path)
```

### 3.2 平台适配层文件

| 文件 | 职责 | 平台 |
|------|------|------|
| `utils/paths.py` | 路径工具（locale检测，输出目录） | 全平台 |
| `utils/platforms/desktop_driver.py` | 桌面应用驱动（窗口管理、截图） | Windows/macOS/Linux |
| `utils/platforms/mobile_driver.py` | 移动端驱动（Appium/XCUITest/UIAutomator） | Android/iOS |
| `utils/platforms/network_throttle.py` | 网络模拟（延迟/丢包/带宽限制） | 全平台 |
| `utils/platforms/blockchain_test.py` | 区块链测试（Ganache/Hardhat） | 全平台 |
| `utils/platforms/iot_helper.py` | IoT设备测试（MQTT/CoAP/BLE） | 全平台 |
| `utils/platforms/media_validator.py` | 媒体文件验证（图像/音频/视频） | 全平台 |
| `utils/platforms/miniprogram_runner.py` | 小程序运行器（微信/支付宝） | Android/iOS |
| `runtime/core/sandbox.py` | 沙箱管理（L1进程隔离按平台） | Windows/macOS/Linux |
| `runtime/backends/` | 7种执行后端（每后端封装平台差异） | 全平台 |
| `runtime/infra/timeout.py` | 超时管理（信号 vs 线程） | 全平台 |
| `runtime/cli/cross_env.py` | CLI跨环境兼容 | 全平台 |

### 4.2 Desktop Release 构建矩阵（`desktop-v2-release.yml`）

```yaml

  matrix:
    include:
      - os: windows-latest       # Windows x86_64
```

沿用 Electron + PyInstaller 架构，按平台分 Job：

| Job | 运行器 | 产物格式 |
|------|--------|---------|
| `build-windows` | windows-latest | `.exe` (NSIS installer) |
| `build-macos` | macos-latest | `.dmg` |
| `build-linux` | ubuntu-latest | `.AppImage` |

### 4.4 平台特定注意事项

**Windows:**
- PyInstaller 构建使用 `--clean --noconfirm`
- Electron Builder 使用 NSIS 打包为 `.exe` 安装程序
- 需要 `windows-latest` runner（当前: Windows Server 2022）

**macOS:**
- 需要 `macos-latest` runner（当前: macOS 14 ARM64）
- Intel 构建目标通过 `target: x86_64-apple-darwin` 交叉编译
- `install.py` 在 macOS 上有专门的端到端测试（`ci.yml` 中的 `macos-real-install`）

**Linux:**
- Tauri 2 构建依赖 `libwebkit2gtk-4.1-dev` 等系统库
- AppImage 构建使用 `patchelf` 修改 rpath
- Docker 构建独立镜像（`runtime/Dockerfile`）不依赖 GitHub Actions runner

## 7. 编码规范（平台相关）

### 7.1 禁止模式

```python
# 禁止 1: 硬编码路径分隔符
path = "workspace\output\report.json"  # 禁止
path = Path("workspace") / "output" / "report.json"  # 正确

# 禁止 2: 平台假设
os.system("clear")   # Windows 上无 clear 命令
os.system("cls" if os.name == "nt" else "clear")  # 或使用 Rich.console.clear()

# 禁止 3: 硬编码临时目录
tmp = "/tmp/test-agent"     # Windows 上可能不存在
tmp = tempfile.gettempdir()  # 正确

# 禁止 4: 假设 shell 可用
subprocess.run("ls -la", shell=True)              # 禁止
subprocess.run(["dir" if os.name=="nt" else "ls", "-la"], shell=False)  # 可用但繁琐
# 正确: 用 pathlib + os 替代 shell 命令
list(Path(".").iterdir())  # 替代 ls
```

### 7.2 推荐模式

```python
# 推荐 1: pathlib 统一路径
from pathlib import Path
work_dir = Path(tempfile.mkdtemp(prefix="test-agent-"))
report = work_dir / "reports" / "result.json"
report.parent.mkdir(parents=True, exist_ok=True)

# 推荐 2: shutil.which 查找可执行文件
python_path = shutil.which("python") or shutil.which("python3")

# 推荐 3: sys.platform 仅在适配层使用
if sys.platform == "win32":
    # Windows 特定代码（仅在 runtime/core/sandbox.py 等适配层）
    ...

# 推荐 4: 使用 os.pathsep 处理路径列表
env_path = os.environ.get("PATH", "")
paths = env_path.split(os.pathsep)
```

## 9. 平台兼容性检查清单

开发新功能时，按此清单验证跨平台兼容：

- [ ] 所有路径使用 `pathlib.Path`，无字符串拼接
- [ ] 子进程使用列表形式参数（`[cmd, arg1, arg2]`），非 `shell=True`
- [ ] 无平台特定 shell 命令（`ls`, `clear`, `cat` -> 用 Python 替代）
- [ ] 临时文件使用 `tempfile` 模块
- [ ] 环境变量名统一大写（`TAGENT_*`）
- [ ] 超时处理兼容 Windows（无 SIGALRM）
- [ ] 文件权限操作有 Windows fallback
- [ ] 行尾符无硬编码（`\r\n` / `\n` 假设）

- [ ] 在 CI `macos-real-install` 中通过测试（macOS 最严格 POSIX 验证）
