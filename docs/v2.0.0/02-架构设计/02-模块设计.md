# V2.0.0 模块设计

> **日期:** 2026-06-21
> **阶段:** 阶段 1：架构设计
> **原则:** 遵循 四-A 5大原则 + 零-C 优先级链
> **上层依赖下层，下层不感知上层；跨层只向下**

## 1. Layer 0 — Runtime Core（核心引擎）

> **职责:** 零依赖上层，纯执行逻辑。不含 AI、LLM、UI 代码。
> **目录:** `runtime/core/`

### 1.1 Sandbox（沙箱管理）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/core/sandbox.py` |
| **职责** | 3级沙箱隔离，防逃逸，提供安全执行环境 |
| **关键类** | `SandboxLevel(Enum)`, `SandboxConfig`, `SandboxResult`, `ProcessSandbox` |
| **工厂函数** | `create_sandbox(level, network) -> ProcessSandbox` |
| **依赖** | `subprocess`, `tempfile`, `pathlib`, `shutil`（标准库，零外部依赖） |

**沙箱分级:**

```
L1 ProcessSandbox:  子进程隔离（默认，C端）
  - CPU: 2核，内存: 256MB，磁盘: 100MB，超时: 300s
  - 工作目录: /tmp/test-agent-{uuid}
  - 网络: 默认关闭，80/443白名单
  - 超时终止: SIGTERM -> 3s -> SIGKILL

L2 ContainerSandbox: Docker/Podman容器（B端）
  - 网络: 白名单
  - 文件系统: tmpfs，退出即销毁
  - 禁止: 宿主机挂载、特权模式

L3 VMSandbox:        Firecracker microVM（最高安全）
  - 完整OS级隔离
```

**接口:**
```python
class ProcessSandbox:
    @property
    def work_dir(self) -> Path: ...
    def execute(self, command, env=None, timeout=None) -> SandboxResult: ...
    def cleanup(self) -> None: ...
    def detect_escape(self) -> bool: ...
```

### 1.2 Backend Adapter（执行后端适配）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/backends/base.py` |
| **职责** | 执行环境抽象，解耦测试逻辑与运行环境 |
| **关键类** | `BaseExecutionEnv(ABC)`, `ExecResult` |
| **注册机制** | `REGISTRY: dict[str, type]` + `@register(name)` 装饰器 |
| **工厂函数** | `get_backend(name, **kwargs) -> BaseExecutionEnv` |
| **依赖** | `abc`, `pathlib`, `dataclasses`（标准库） |

**7种后端实现:**

| 后端 | 文件 | 场景 |
|------|------|------|
| `local` | `runtime/backends/local.py` | 本机subprocess直跑 |
| `docker` | `runtime/backends/docker.py` | 容器隔离 |
| `ssh` | `runtime/backends/ssh.py` | 远程主机长连接(paramiko) |
| `singularity` | `runtime/backends/singularity.py` | HPC集群(SLURM友好) |
| `modal` | `runtime/backends/modal.py` | Serverless持久化，闲置hibernate |
| `daytona` | `runtime/backends/daytona.py` | Serverless dev sandbox |
| `vercel_sandbox` | `runtime/backends/vercel_sandbox.py` | 边缘运行 |

**接口契约（保持现有，V2.0.0冻结）:**
```python
class BaseExecutionEnv(ABC):
    name: str = "abstract"
    async def connect(self) -> None: ...
    async def exec(self, cmd, *, timeout=60.0, cwd=None, env=None) -> ExecResult: ...
    async def read(self, path) -> bytes: ...
    async def write(self, path, data) -> None: ...
    async def sync_in(self, local, remote) -> None: ...
    async def sync_out(self, remote, local) -> None: ...
    async def close(self) -> None: ...
```

## 2. Layer 1 — Agent Layer（智能层）

> **职责:** 所有 AI/LLM 相关逻辑，不依赖具体存储/通知/界面。
> **目录:** `runtime/agent/`, `runtime/orchestrator/`, `runtime/router/`

### 2.1 Permission Manager（权限管理）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/agent/permissions.py` |
| **职责** | 3级权限: allow/deny/ask + glob模式匹配 + 用户确认 |
| **关键类** | `PermissionLevel(Enum)`, `PermissionRule`, `PermissionDecision`, `PermissionManager` |
| **接口** | `check(operation, target) -> PermissionDecision` |
| **依赖** | `fnmatch`, `rich.prompt`（标准库 + Rich） |

**内置规则:**
- **DENY（9条）:** `rm -rf /*`, `sudo *`, 写入 `.env`/`id_rsa*`, fork bomb 等
- **ALLOW（8条）:** `Read(**/*.py)`, `Write(workspace/**)`, `Bash(git *)`, `Bash(pytest *)` 等

### 2.2 Prompt Guard（注入防御）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/agent/prompt_guard.py` |
| **职责** | 6层纵深防御: 输入隔离 -> 输出校验 -> 敏感操作确认 -> 角色隔离 -> 输入净化 -> 审计日志 |
| **关键函数** | `sanitize_input(text) -> SanitizationResult`, `is_destructive_operation(command) -> bool` |
| **依赖** | `re`, `unicodedata`（标准库） |

**防御层次:**
- L1 输入隔离: 用户输入标记为 USER 角色，永不SYSTEM
- L2 输出校验: LLM工具调用执行前校验
- L3 敏感确认: 破坏性操作必须二次确认
- L4 角色隔离: 3层不可变角色（核心规则 > 任务 > 上下文）
- L5 输入净化: 控制字符 + 同形字检测 + 长度限制（32KB）
- L6 审计日志: 记录所有被拦截的注入尝试

## 3. Layer 2 — Infrastructure（基础设施）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/config/settings.py` + `runtime/cli/config.py` |
| **职责** | 5层配置优先级: CLI参数 > 环境变量 > 项目配置(tagent.yaml) > 用户配置 > 默认值 |

用户 ~/.tagent/config.yaml  <- 用户级
默认值（代码内定义）          <- 兜底

### 3.3 Storage（存储策略）

| 属性 | 内容 |
|------|------|
| **文件** | `runtime/infra/storage_strategy.py` |
| **职责** | 3级分层存储: 热（内存+WAL）-> 温（SQLite/PostgreSQL）-> 冷（JSON文件/对象存储） |
| **关键类** | `TieredStorage`, `StorageTier(Enum)`, `RetentionPolicy`, `StorageStats` |

**数据保留策略:**

| 层级 | 保留期 | 上限 | 存储 |
|------|--------|------|------|
| Hot | 1天 | 10000条 | 内存 + WAL |
| Warm | 30天 | 10000条 | SQLite(社区) / PostgreSQL(企业) |
| Cold | 365天 | 无限制 | JSON文件 / 对象存储 |

**接口:**
```python
class TieredStorage:
    def hot_put(key, data) -> None: ...
    def hot_get(key) -> dict | None: ...
    def warm_put(key, data) -> None: ...
    def warm_get(key) -> dict | None: ...
    def archive(key) -> None: ...
    def cleanup_expired() -> int: ...
    def stats() -> StorageStats: ...
    def export_json(output_path) -> None: ...
    def export_csv(output_path, fields) -> None: ...
```

## 8. 跨模块依赖规则

> **遵循 四-A 五大原则，其中原则 2（单向依赖）为本文档核心约束。**

### 8.1 依赖方向（严格单向，不可反向）

```
Layer 3 (Interface)        --向下-->  Layer 2 (Infrastructure)
Layer 3 (Interface)        --向下-->  Layer 1 (Agent Layer)
Layer 3 (Interface)        --向下-->  Layer 0 (Core)
Layer 1 (Agent Layer)      --向下-->  Layer 0 (Core)
Layer 2 (Infrastructure)   --向上-->  Layer 1 (Agent Layer)    <- 禁止
Layer 2 (Infrastructure)   --向下-->  Layer 0 (Core)           <- 允许
Layer 0 (Core)             --向上-->  任何层                    <- 禁止
Layer 4 (Packaging)        --运行时--> 任何层                   <- 禁止（Build-time）
```

### 8.2 跨层调用允许

- Layer 3 可以直接调用 Layer 0（跳过 Layer 1/2），但推荐经过 Layer 1
- Layer 1 不能调用 Layer 2 的存储/通知功能（如需持久化，通过 Layer 0 工作目录）

### 8.3 横向调用规则

- 同层模块之间通过接口调用，不应直接导入实现
- 跨层通过 ExecutionContext 对象传递 Trace ID、LLM Provider 等上下文
- 全局单例（PermissionManager、HookRegistry、TelemetryManager）通过 `get_*()` 工厂函数获取

### 8.4 测试依赖

- 每个模块必须有对应的 Fake 实现（见 `runtime/infra/di.py`）
- 单元测试不使用真 LLM（使用 `FakeLLM` 或 `TAGENT_LLM_PROVIDER=stub`）
- 集成测试遵循 mock 边界：只 mock 外部系统（API、DB），不 mock 内部模块

### 8.5 新增模块检查清单

1. 文件放在正确的 Layer 目录
2. 仅导入同层或下层模块
3. 提供 Protocol / ABC 接口
4. 提供 Fake 实现用于测试
5. 注册到对应 INDEX.md
6. 依赖声明在 `requirements.txt` 或 `pyproject.toml`
