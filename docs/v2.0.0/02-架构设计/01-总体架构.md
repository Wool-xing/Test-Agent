## 1. 5层分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│ Layer 3 — Interface (用户界面，不包含业务逻辑)                      │
│  CLI REPL (Typer) │ TUI (Textual) │ Web Dashboard (React)         │
│  API Server (FastAPI) │ IM Bots (企业微信/飞书/钉钉)               │
├─────────────────────────────────────────────────────────────────┤
│ Layer 2 — Infrastructure (基础设施，不依赖具体界面)                  │
│  Config Manager │ Logger (Loguru) │ Storage (SQLite/PG)            │
│  Scheduler (Cron) │ Task Queue │ Notification (Slack/Email/IM)     │
│  Trace/Corelation ID │ Telemetry (OpenTelemetry)                   │
├─────────────────────────────────────────────────────────────────┤
│ Layer 1 — Agent Layer (智能层，不依赖具体存储/通知)                  │
│  LLM Providers (Claude/GPT/Gemini/Ollama) │ Tool System            │
│  Hook System │ Skill Registry │ Permission Manager                  │
│  Agent Orchestrator │ Router/Intent Detection                      │
│  Prompt Manager │ Context/Compaction Manager                       │
├─────────────────────────────────────────────────────────────────┤
│ Layer 0 — Runtime Core (核心引擎，零依赖上层)                       │
│  Test Executor │ Assertion Engine │ Result Collector               │
│  Sandbox Manager │ Execution Backend Adapter                       │
│  Data Manager (fixtures/generators/cleanup)                        │
├─────────────────────────────────────────────────────────────────┤
│ Layer 4 — Packaging (打包分发，Build-time)                          │
│  PyInstaller (CLI binary) │ Docker Image                            │
│  npm/pip/brew/Scoop │ APK/IPA (Flutter/RN)                         │
└─────────────────────────────────────────────────────────────────┘
```

### 依赖方向（严格单向，不可反向）

```
Layer 3 → Layer 2 → Layer 1 → Layer 0
Layer 3 → Layer 1 → Layer 0  (可跨层向下)
Layer 0 → Layer 1              (禁止向上)
Layer 1 → Layer 2              (禁止向上)

Layer 4 是特例 —— Build-time层，不参与运行时依赖
```

## 2. 模块划分与职责

### Layer 0 — Runtime Core

| 模块 | 路径 | 职责 | 接口 |
|------|------|------|------|
| Test Executor | `runtime/core/executor.py` | 执行测试逻辑，与LLM/UI无关 | `execute(test_def, ctx) → TestResult` |
| Assertion Engine | `runtime/core/assertion.py` | 断言库（等值/范围/模式/性能/结构） | `assert_*(actual, expected, config)` |
| Result Collector | `runtime/core/collector.py` | 聚合测试结果，流式输出 | `collect(results) → TestReport` |
| Sandbox Manager | `runtime/core/sandbox.py` | 沙箱分级管理（进程/容器/VM） | `create_sandbox(level) → SandboxContext` |
| Backend Adapter | `runtime/core/backend.py` | 执行环境抽象（local/docker/ssh） | `BaseExecutionEnv` (现有接口保持) |
| Data Manager | `runtime/core/data.py` | 测试数据生命周期管理 | `setup_fixtures() → cleanup()` |

### Layer 1 — Agent Layer

| 模块 | 路径 | 职责 | 接口 |
|------|------|------|------|
| LLM Providers | `runtime/agent/llm/` | 多Provider抽象（Claude/GPT/Gemini/Ollama） | `LLMProvider.complete(prompt) → str` |
| Tool System | `runtime/agent/tools/` | 内置工具（文件/Shell/网络/数据库） | `Tool.execute(params) → ToolResult` |
| Hook System | `runtime/agent/hooks/` | 生命周期Hook（PreToolUse/PostToolUse/Stop） | `HookRegistry.register(event, handler)` |
| Skill Registry | `runtime/agent/skills/` | Skill注册/发现/加载/生命周期 | `SkillRegistry.register(name, skill)` |
| Permission Manager | `runtime/agent/permissions/` | 3级权限（allow/deny/ask）+ 路径匹配 | `check_permission(op, target) → bool` |
| Agent Orchestrator | `runtime/agent/orchestrator.py` | 主Agent→子Agent编排（单/并行/Pipeline/Socratic） | `orchestrate(task, mode) → AgentResult` |
| Router | `runtime/agent/router.py` | 意图检测 + Skill路由 + LLM决策 | `route(artifact) → RoutingDecision` |
| Prompt Manager | `runtime/agent/prompts.py` | System Prompt模板管理 + 反漂移注入 | `assemble_prompt(task, context) → str` |
| Context Manager | `runtime/agent/context.py` | Token预算 + Compaction + 优先级 | `manage_context(session) → ContextState` |

### Layer 2 — Infrastructure

| 模块 | 路径 | 职责 | 接口 |
|------|------|------|------|
| Config Manager | `runtime/infra/config.py` | 5层配置（CLI > Env > Project > User > Default） | `get_config() → Settings` |
| Logger | `runtime/infra/logger.py` | 结构化日志（JSON）+ 轮转 + 脱敏 + Trace ID | `logger.info(msg, trace_id=...)` |
| Storage | `runtime/infra/storage.py` | SQLite(C端) + PostgreSQL(B端) 统一接口 | `StorageAdapter.execute(query)` |
| Scheduler | `runtime/infra/scheduler.py` | Cron解析 + 任务队列 + 重试 + 死信队列 | `schedule(cron_expr, task) → JobId` |
| Notification | `runtime/infra/notify.py` | Slack/Email/微信/飞书/钉钉 统一Notifier接口 | `Notifier.send(msg, channel)` |
| Telemetry | `runtime/infra/telemetry.py` | OpenTelemetry trace/metric/log 导出 | `tracer.start_span(name)` |

### Layer 3 — Interface

| 模块 | 路径 | 职责 | 接口 |
|------|------|------|------|
| CLI | `runtime/ui/cli/` | Typer CLI + 自然语言REPL | `tagent [command]` |
| TUI | `runtime/ui/tui/` | Textual 10面板完整TUI | `tagent tui` |
| Web API | `runtime/ui/web/` | FastAPI REST + WebSocket | `/api/v1/*` |
| Web Dashboard | `apps/desktop-v2/` | React + Vite 管理面板 | 浏览器访问 |
| IM Bots | `runtime/ui/bots/` | 企业微信/飞书/钉钉 Bot | Webhook接收 |
| Mobile | `apps/mobile/` | Flutter/React Native APK/IPA | `tagent mobile` |

## 3. 核心接口契约

### 3.1 AgentRunner（Layer 1）
```python
class AgentRunner(ABC):
    """所有Agent的基类 — 冻结于V2.0.0"""
    @abstractmethod
    def system_prompt(self) -> str: ...
    @abstractmethod
    def user_prompt(self, ctx: RunnerContext) -> str: ...
    def run(self, ctx: RunnerContext) -> RunnerResult: ...
```

### 3.2 SkillProtocol（Layer 1）
```python
class SkillProtocol(Protocol):
    """Skill接口 — 第三方开发者实现此接口"""
    name: str
    permissions: SkillPermissions
    def execute(self, params: SkillParams, ctx: ExecutionContext) -> SkillResult: ...
    def validate(self, params: dict) -> bool: ...
```

### 3.3 BaseExecutionEnv（Layer 0）
```python
class BaseExecutionEnv(ABC):
    """执行后端抽象 — 保持现有接口"""
    def connect(self) -> bool: ...
    def exec(self, command: str, timeout: int) -> ExecResult: ...
    def close(self) -> None: ...
```

### 3.4 ToolProtocol（Layer 1）
```python
class ToolProtocol(Protocol):
    """工具接口 — MCP兼容"""
    name: str
    description: str
    parameters: dict  # JSON Schema
    def execute(self, params: dict) -> ToolResult: ...
```

### 3.5 NotifierProtocol（Layer 2）
```python
class NotifierProtocol(Protocol):
    """通知渠道接口"""
    channel: str
    def send(self, message: str, target: str) -> bool: ...
```

## 4. 数据流

### 4.1 自然语言测试执行（主流程）

```
用户输入 "检查 www.example.com 是否正常"
    │
    ▼
Layer 3 — CLI REPL 接收
    │
    ▼
Layer 1 — Router 意图检测 → RoutingDecision {skill: "http_check", params: {url: "www.example.com"}}
    │
    ▼
Layer 1 — Permission Manager 检查权限
    │  (允许 → 继续 / 拦截 → TUI弹窗确认)
    ▼
Layer 1 — Skill Registry 查找 http_check Skill
    │
    ▼
Layer 0 — Sandbox Manager 创建沙箱（级别1: 进程隔离）
    │
    ▼
Layer 0 — Test Executor 执行 http_check
    │
    ▼
Layer 0 — Result Collector 收集结果
    │
    ▼
Layer 1 — LLM Provider 生成自然语言报告
    │
    ▼
Layer 3 — TUI 执行面板显示结果
```

### 4.2 CI/CD流水线执行

```
GitHub Actions 触发
    │
    ▼
Layer 3 — CLI: tagent run --ci --format junit
    │
    ▼
Layer 2 — Config Manager 加载 CI 模式配置
    │
    ▼
Layer 1 — Agent Orchestrator 按 DAG 顺序执行
    │
    ▼
Layer 0 — 并行测试执行（Worker Pool）
    │
    ▼
Layer 2 — Logger 写结构化日志 + Trace ID
    │
    ▼
Layer 3 — CLI 输出 JUnit XML → CI artifact
```

### 4.3 交互式TUI流程

```
用户启动 tagent tui
    │
    ▼
Layer 3 — TUI 渲染10面板布局
    │
    ▼
对话面板: 用户输入自然语言
    │
    ▼
Layer 1 — Agent 处理（同4.1）
    │
    ▼
执行面板: 实时进度更新（WebSocket推送）
    │
    ▼
报告面板: 历史趋势图
    │
    ▼
Layer 2 — Storage 持久化会话
```

## 5. B端 / C端 架构分离

```
                    ┌──────────────────┐
                    │   Core Engine     │
                    │  (Layer 0 + 1)    │
                    │  完全共享，不分叉  │
                    └────────┬─────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
      ┌───────┴───────┐             ┌───────┴───────┐
      │  Community Mode │             │ Enterprise Mode│
      │  (tagent.yaml:   │             │ (tagent.yaml:   │
      │   mode: community│             │  mode: enterprise│
      ├─────────────────┤             ├─────────────────┤
      │ 认证: 本地/GitHub│             │ 认证: SSO/LDAP   │
      │ 存储: SQLite     │             │ 存储: PostgreSQL │
      │ 部署: 一键脚本   │             │ 部署: Docker/K8s │
      │ 协作: 单人       │             │ 协作: 团队/RBAC  │
      │ UI: CLI+TUI为主  │             │ UI: Web Dashboard│
      │ 定价: 免费开源   │             │ 定价: License    │
      │ 支持: GitHub Issue│            │ 支持: 工单+SLA   │
      └─────────────────┘             └─────────────────┘
```

**实现方式：** 单仓库，`tagent.yaml` 中 `mode` 字段切换。
- 核心引擎不分叉（Layer 0+1完全共享）
- Layer 2 中 Config/Storage/Auth 按模式加载不同实现
- Layer 3 中 UI 按模式显示不同界面

## 6. 多平台适配策略

| 平台 | 关键差异 | 处理方式 |
|------|---------|---------|
| Windows | 路径分隔符 `\`；无内置bash；无Unix信号 | `pathlib` 统一路径；内嵌bash(busybox)；Win32 API替代信号 |
| macOS | Homebrew；Apple公证；Keychain | 构建`.dmg`；`codesign`签名；Keychain存储Token |
| Linux | 多发行版；systemd；AppArmor | `.deb`/`.rpm`/AppImage；systemd service；seccomp沙箱 |
| Android | 受限文件系统；无Python运行时 | Flutter APK；Chaquopy嵌入Python |
| iOS | 沙箱；无JIT；无子进程 | Flutter IPA；Python受限子集 |
| Web | 浏览器沙箱；无文件系统 | PWA；IndexedDB存储；Service Worker离线 |

**平台差异集中在：** `runtime/core/platform/` — 一个适配层，不在业务逻辑中散落 `if windows/elif mac`。

## 7. 扩展机制设计

### 7.1 Skill（轻量扩展 — 1个SKILL.md + 可选脚本）
```
skills/{name}/
├── SKILL.md       ← 入口（YAML元数据 + 参数表 + 输出格式 + 自带测试）
├── executor.py    ← 执行逻辑
├── schema.py      ← 输入/输出 Schema
└── test_{name}.py ← 自带测试（≥3真实断言）
```

### 7.2 Plugin（重量扩展 — Python包，实现接口）
```
plugins/{name}/
├── setup.py
├── {name}/
│   ├── __init__.py
│   ├── provider.py   ← 实现 ToolProtocol / NotifierProtocol 等
│   └── schema.py
```

### 7.3 MCP Server（外部工具集成）
- 配置在 `tagent.yaml` → `mcp_servers` 段
- 自动发现工具
- MCP工具与内置工具在Tool System中统一管理
