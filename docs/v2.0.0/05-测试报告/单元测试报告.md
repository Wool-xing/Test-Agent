## 五、失败与跳过详情

### 5.1 失败用例 (1 个)

| 用例 | 文件 | 原因 |
|------|------|------|
| `TestPingCheck::test_ping_script_cli` | `test_basic_skills.py` | Windows GBK 编码问题: subprocess 读取 ping 输出时 `UnicodeDecodeError: gbk codec cannot decode byte 0xae` |

**根因分析:** `test_ping_script_cli` 通过 subprocess 调用 Windows ping 命令，在中文 Windows 环境下 ping 输出包含 GBK 编码字符。subprocess 的 reader thread 使用默认 GBK 解码时遇到无法解码的字节，触发 `UnicodeDecodeError`。

**影响范围:** 仅影响中文 Windows 平台。Linux/macOS 的 ping 输出为纯 ASCII，不受影响。

**修复建议:** 在 subprocess 调用中显式指定 `encoding='utf-8'` 并设置 `errors='replace'`，或在测试中 mock subprocess 调用避免平台差异。

### 5.2 跳过用例 (2 个)

| 用例 | 文件 | 跳过条件 |
|------|------|---------|
| Impact Engine 相关测试 (2个) | `test_impact_engine.py` | `graph.json` 不存在 -> 降级模式 (`pytest.skip()`) |

**说明:** Impact Engine 依赖 `graphify-out/graph.json` 知识图谱文件。当前环境未运行过 `/graphify --update`，图谱文件不存在。测试通过 `pytest.skip()` 优雅降级，不影响其他测试。
