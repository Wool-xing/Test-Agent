## 8. `workspace/测试报告/{项目名}/baselines/perf_baseline.json`

性能基线文件，**首次自动生成**。生成条件：

- release 分支
- `PERF_MODE=full`
- 当次门禁全部 PASS

手动触发更新：

```bash

python -m utils.jmeter_result_parser \
    workspace/测试报告/{项目名}/jmeter-results/result.jtl \
    --mode full \
    --baseline workspace/测试报告/{项目名}/baselines/perf_baseline.json \
    --update-baseline

```text

文件结构：

```json

{
  "total_requests": 15000,
  "tps": 102.5,
  "avg_response_ms": 145,
  "p95_response_ms": 420,
  "p99_response_ms": 680,
  "error_rate_pct": 0.12,

}

## 9. 通知 webhook 申请教程

### 企业微信群机器人

1. 群聊 → 右上角 `...` → 群机器人 → 添加机器人

## 11. 常见配置错误

| 错误 | 原因 | 解决 |
| ------ | ------ | ------ |
| `TEST_ENV=prod` 启动报 ValueError | get_current_env 直接 raise（防误测生产） | 改为 test / staging |
| 覆盖率永远 0% | cov 指向了测试脚本本身 | 设置 `APP_SRC_PATH=./src` |
| pytest 报 `Unknown marker: api` | strict-markers + 未注册 | 在 pytest.ini `markers` 段补 |
| 性能门禁永远不通过 | CI 用 5 并发但门禁 TPS≥100 | `PERF_MODE=ci_quick`（CI 默认即此值） |
| GitHub Actions if 引用 secrets 永远 false | secrets 不能直接 if | yml 已用 outputs 中转模式（preflight job） |
| Jenkins JMeter 找不到命令 | sh 块间 PATH 不持久 | `withEnv(["PATH+JMETER=..."])` 持久 PATH |
| 禅道 Bug 提交 401 | token 过期 | utils/zentao_bug_manager 自动续期，重新调用即可 |

## 12. 依赖版本管理

### 策略：固定版 + Dependabot 自动 PR + CVE 拦截

| 层级 | 策略 |
| ------ | ------ |
| 当前 `requirements.txt` | `==` 锁版本，保 CI 可复现 |
| Dependabot 周扫描 | `.github/dependabot.yml` 每周一 9:00 自动开 PR（北京时间） |
| CI 拦截 CVE | `.github/workflows/test.yml` 跑 `pip-audit` + `safety`，发现高危直接 fail |
| 季度评审 | 详见 `部署说明.md` "依赖升级管理" 段 SOP |

### Dependabot 分组

`.github/dependabot.yml` 把相关库分组，避免 PR 爆炸：

- `pytest-ecosystem`（pytest + pytest-* + allure-pytest）
- `security-tools`（bandit / safety）
- `ui-automation`（playwright / Appium / pywinauto / selenium）
- `performance`（locust）
- `data-doc`（openpyxl / docx / pptx / reportlab / pdf / yaml / bs4）
- `database`（psycopg2 / pymysql / SQLAlchemy）
- `protocols`（requests / websocket / mqtt / kafka / paramiko / pyserial）
- `ai-ml`（scikit-learn / scipy / faker / factory-boy）

### 锁定（禁止 Dependabot 升 major）

- `pytest`（7.x → 8.x 需手动评估，破坏性变更）
- `playwright`（major 升级 API 兼容性敏感）

按需追加：编辑 `.github/dependabot.yml` `ignore` 段。

### 常用命令

```bash

# 看可升列表

pip list --outdated

# CVE 扫描（CI 已自动跑）

pip-audit -r requirements.txt --strict
safety check -r requirements.txt

# 一次性升所有（慎用）

pip install -U -r requirements.txt

```text
