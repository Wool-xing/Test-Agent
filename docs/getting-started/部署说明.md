# 部署说明

## 系统要求

| 组件 | 最低版本 | 推荐版本 | 必需 |
|------|---------|---------|------|
| Claude Code CLI | 1.0+ | 最新版 | ✅ |
| Python | 3.10 | 3.11 | ✅ |
| Node.js | 18 | 20 LTS | ✅（Claude Code 依赖） |
| Java JRE | 8 | 17 | ✅（JMeter 依赖） |
| Apache JMeter | 5.6.3 | 5.6.3 | ✅（性能测试） |
| Allure CLI | 2.27+ | 2.29 | ✅（HTML 报告） |
| Git | 2.30 | 最新 | ✅ |
| Docker | 20.10+ | 最新 | ⚪ 可选（mock-server） |

## 路径约定

部署涉及两个目录：

- `TEMPLATE_DIR`：模板源目录（本仓库克隆到本地的位置，路径自定义）
- `PROJECT_ROOT`：用户的测试项目目录

部署脚本通过参数/环境变量传入，**不再硬编码任何具体路径**。

### 步骤 2：安装 JMeter

```powershell
$JMETER_VERSION = "5.6.3"
$JMETER_HOME = "C:\Tools\apache-jmeter-$JMETER_VERSION"

Invoke-WebRequest -Uri "https://archive.apache.org/dist/jmeter/binaries/apache-jmeter-$JMETER_VERSION.zip" `
    -OutFile "$env:TEMP\jmeter.zip"
```

## macOS / Linux 部署（一键脚本）

```bash
#!/bin/bash
# deploy.sh
set -euo pipefail

TEMPLATE_DIR="${TEMPLATE_DIR:-$(pwd)}"
PROJECT_ROOT="${1:-$(pwd)/test-project}"

echo "=========================================="
echo " Test-Agent 工作流部署 V2.0.0"
echo " 模板目录: $TEMPLATE_DIR"
echo " 项目目录: $PROJECT_ROOT"
echo "=========================================="

# ===== 1. 检查基础工具 =====
need() { command -v "$1" >/dev/null 2>&1 || { echo "❌ 缺少 $1"; exit 1; }; }
need python3
need node
need npm
need git
need java || echo "⚠️  Java 未安装，JMeter 性能测试不可用"

# ===== 2. 安装 Claude Code =====
if ! command -v claude >/dev/null 2>&1; then
    echo "正在安装 Claude Code..."
    npm install -g @anthropic-ai/claude-code
fi

# ===== 3. 创建目录结构 =====
mkdir -p "$PROJECT_ROOT"/.claude/{agents,skills}
mkdir -p "$PROJECT_ROOT"/.github/workflows

mkdir -p "$PROJECT_ROOT"/workspace/测试报告/{项目名}/{allure-results,jmeter-results,jmeter-report,coverage-report,baselines,history,截图,报告}

# ===== 4. 拷贝 Agent / Skill 定义（显式列名）=====
for f in 01-测试主管 02-需求分析 03-用例设计 04-环境管理 05-数据准备 06-自动化脚本 07-测试执行 08-Bug管理 09-报告生成 10-移动测试 11-桌面测试 12-视觉游戏测试 13-系统集成测试 14-AI模型测试 15-渗透测试 16-车载测试; do

cp "$TEMPLATE_DIR/config/pytest.ini"       "$PROJECT_ROOT/"
cp "$TEMPLATE_DIR/config/.mcp.json"        "$PROJECT_ROOT/"
cp "$TEMPLATE_DIR/config/requirements.txt" "$PROJECT_ROOT/"
[[ -f "$PROJECT_ROOT/.env" ]] || cp "$TEMPLATE_DIR/config/.env.example" "$PROJECT_ROOT/.env"

# ===== 6. utils（含新增 8 个 + 现有 3 个 + __init__）=====
for f in __init__.py api_retry_util.py data_factory.py data_masking.py \
         excel_generator.py flaky_detector.py generate_report.py \
         jmeter_csv_exporter.py jmeter_result_parser.py \
         regression_scope.py zentao_bug_manager.py ci_quality_gate.py \
         mobile_driver.py miniprogram_runner.py desktop_driver.py \
         visual_helper.py iot_helper.py media_validator.py \
         tracing_validator.py mq_helper.py ai_validator.py \
         prd_loader.py websocket_helper.py protocol_helper.py \
         security_scanner.py network_throttle.py chaos_helper.py \
         soak_runner.py ux_metrics.py compatibility_matrix.py \
         state_machine_tester.py pairwise_generator.py bdd_runner.py \
         web_vitals_collector.py api_security_scanner.py fuzzer.py \
         db_test_helper.py contract_test.py openapi_test_gen.py \
         push_test.py a11y_scanner.py i18n_checker.py \
         mutation_runner.py dora_metrics.py blockchain_test.py ai_adversarial.py \
         slo_validator.py email_sender.py suite_minimizer.py; do
    cp "$TEMPLATE_DIR/utils/${f}" "$PROJECT_ROOT/utils/"
done

# ===== 7. CI/CD =====
cp "$TEMPLATE_DIR/ci/github-actions-test.yml" "$PROJECT_ROOT/.github/workflows/test.yml"
cp "$TEMPLATE_DIR/ci/jenkins-pipeline.groovy" "$PROJECT_ROOT/Jenkinsfile"

# ===== 8. Python 虚拟环境 =====
cd "$PROJECT_ROOT"
if [[ ! -d ".venv" ]]; then
    python3 -m venv .venv
fi

# fail-fast: 必须能成功 source venv，否则退出

source .venv/bin/activate

echo ""
echo "=========================================="
echo " ✅ 部署完成"
echo ""
echo " 下一步："
echo " 1. 编辑 $PROJECT_ROOT/.env 填写实际配置"
echo " 2. 安装 JMeter 5.6.3（见部署说明 'JMeter 安装'）"
echo " 3. 安装 Allure CLI（brew install allure 或 scoop install allure）"
```

## 依赖升级管理（Dependabot + 季度 SOP）

### 自动化（已配 `.github/dependabot.yml`）

GitHub 仓库会**每周一 9:00**（北京时间）自动扫描 + 开 PR：
- pip 依赖（按 pytest / 安全 / UI / 数据 / DB / 协议 / AI 分组）
- GitHub Actions 版本
- npm 包

### CI 自动 CVE 拦截

`.github/workflows/test.yml` 在 code-quality job 跑 `pip-audit + safety`，发现 CVE 直接 fail CI。

### 季度升级 SOP（人工评审 Dependabot PR）

```text
季度第 1 周（patch 升级）：
  □ Dependabot 累积的 patch PR（如 1.0.x → 1.0.y）批量 merge
  □ 跑 /smoke-test 验证

季度第 4-6 周（minor 升级）：
  □ 评估 minor 升级（如 1.x → 1.y），优先评估稳定层
  □ 跑 /regression-test 全量验证
  □ 不通过 → close PR 加注释，等下季度

季度第 10-12 周（major 升级）：
  □ Major 版本（如 1.x → 2.0）单独立项
  □ 开 feature 分支验证 + 性能基线对比
  □ 通过 → merge + 同步基线

紧急 CVE：
  □ pip-audit 标 CRITICAL/HIGH 立即处理
  □ merge → 跑 smoke → 部署
```

### 手动检查命令

```bash
# 看哪些可升
pip list --outdated

# 看 CVE
pip-audit -r requirements.txt --strict --format columns

# 强制全部升级（慎用，须先备份）
pip install -U -r requirements.txt
```

### 锁死特定库（防 Dependabot 升）

`.github/dependabot.yml` 已配置：
- `pytest` major 版禁升（如 7.x → 8.x 需手动）
- `playwright` major 版禁升（API 兼容性敏感）

按需在 `dependabot.yml` `ignore` 段追加。

## 升级 / 回滚 / 卸载

### 升级模板

```bash
cd /path/to/Test-Agent
git pull
TEMPLATE_DIR=$(pwd) ./docs/getting-started/deploy.sh /path/to/your-test-project
```

> 升级会**覆盖** `.claude/agents/`、`.claude/skills/`、`utils/`、`conftest.py`、`pytest.ini`、`requirements.txt`、`.mcp.json`、`.github/workflows/test.yml`、`Jenkinsfile`、`CLAUDE.md`、`AGENTS.md`、`templates/`。
> **不会覆盖**：`.env`、`workspace/`、`src/`、`quality_gates.yaml`。

### 回滚

```bash
cd your-test-project
git checkout HEAD~1 -- .claude utils conftest.py pytest.ini  # 若已 git 管理
pip install -r requirements.txt
```

### 卸载

```bash
cd your-test-project
rm -rf .claude utils conftest.py pytest.ini .mcp.json
```

## 常见问题

**Q: Windows 执行脚本报"无法加载 xxx，因为在此系统上禁止运行脚本"？**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**Q: Playwright 安装慢？**
```bash
# 国内镜像
export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright
playwright install chromium
```

**Q: pip 安装慢？**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**Q: JMeter 5.6.3 下载链接失效？**
- Apache `downloads.apache.org` 只保留最新若干版本
- 历史版本统一从 `archive.apache.org/dist/jmeter/binaries/` 下载
- 部署脚本默认使用 archive 链接（更稳定）

**Q: 如何验证安装是否成功？**
```bash
claude --version             # Claude Code
python --version             # ≥3.10
java -version                # ≥8（JMeter 依赖）
jmeter --version             # 5.6.3
allure --version             # ≥2.27
pytest --collect-only        # 用例收集正常
python -c "from utils.api_retry_util import call_with_retry; print('utils OK')"
```

**Q: ANTHROPIC_API_KEY 配置在哪？**
- 通过 `claude /login` 交互式登录（推荐）
- 或通过 Anthropic 控制台生成 API key 后由 Claude Code 自身管理
- **不要写入 .env**（避免误提交到 Git）

**Q: 4 个无实现的 mcp_server（zentao/wechat/feishu/dingtalk）能用吗？**
- **当前不可用**。`.mcp.json` 仅启用 filesystem
- 通知（企微/飞书/钉钉）走 webhook 直连：`utils/generate_report.send_*`
- Bug 提交走 SDK 直连：`utils/zentao_bug_manager.ZentaoBugManager`
- 如需启用 MCP 通道，自行实现对应 mcp_server 模块后追加配置（见 `.mcp.json` `_comment`）

## 运行时层 `runtime/` 部署

> 运行时层是可选的(用户也可只用 16 专家+32 Skill 的工作流模式)。
> 想要 HTTP/CLI 一键跑、AI 路由、飞轮存储,启它。

### 1. 起本地依赖(Docker)

```bash
cd runtime
docker compose up -d                              # postgres + minio + prefect-server
docker compose --profile observability up -d      # 可选:加 tempo + loki + grafana
```

服务端口:
- Postgres: `5432`
- MinIO: `9000`(API) / `9001`(Console)
- Prefect: `4200`
- Grafana: `3000`(observability profile)

### 2. 装 runtime Python 依赖

```bash
pip install -e ./runtime           # 含 litellm/prefect/fastapi/typer/sqlalchemy/alembic/pgvector/minio/opentelemetry/loguru/pypdf/python-docx/pyyaml/rich

# 或最小集(无 prefect,Direct 执行器即可)
pip install loguru pyyaml pydantic pydantic-settings sqlalchemy fastapi uvicorn typer rich pypdf python-docx litellm minio
```

### 3. 配置(`.env` 加段)

```ini
# === Test-Agent Runtime ===

TAGENT_DB_URL=postgresql+psycopg://tagent:tagent@localhost:5432/tagent
```

### 故障排查

| 现象 | 原因 | 修 |
|------|------|----|
| `litellm not installed` | 仅装最小集 | `pip install litellm` |
| `prefect not installed` | 同上 | 仅最小集时 Direct 自动接管,**不影响功能** |
| `minio bucket not exists` | minio-init 容器未起 | `docker compose up minio-init` |
| `psycopg connection refused` | postgres 未起或端口冲突 | `docker compose ps` 查状态 |
| stub provider 路由不准 | stub 是关键字简陋匹配 | 真模型 ≥85%,M2 双模型投票 |
